## `> PHASE 1 · SYSTEMS & NETWORK DEFENSE`
### `// March – May · Mastering the foundational layers of the tech stack`

<details>
<summary><b>🖥️ Domain 1 · Linux Systems Administration & Security Hardening</b></summary>

<br/>

> *"You can't defend what you don't understand. Week 1 starts at the shell."*

**Core Competencies:** CLI proficiency · FHS navigation · ACL management

| Week | Focus | Artifact |
|:---|:---|:---|
| **W1 · Terminal Supremacy** | Shell navigation, RWX permissions, stream editing with `grep/sed/awk` | `bash_oneliner_challenges.sh` |
| **W2 · Network Foundations** | OSI Model, TCP/IP, Subnetting (CIDR), Protocol Interrogation | `network_audit.txt` · `subnet_blueprint.txt` · `protocol_audit.txt` |
| **W4 · Infrastructure Hardening** | Virtualization (Type 1/2 Hypervisors), Docker, Docker Compose | `sandbox_report.txt` · `deploy_web.sh` · `docker-compose.yml` |
| **W5 · Identity Principles** | User/Group Lifecycle, MFA, SSO, AWS IAM Console | `security_policy_draft.md` |

**Assessment:** Timed system recovery and multi-user permission remediation.

</details>

<details>
<summary><b>🐍 Domain 2 · Programming for Security Automation</b></summary>

<br/>

> *"Manual is a liability. Automation is the weapon."*

**Core Competencies:** Python scripting · log parsing · automated anomaly detection

| Week | Focus | Artifact |
|:---|:---|:---|
| **W3 · Python as a Security Instrument** | Data types, loops, functions, file I/O | `port_check.py` · `log_filter.py` · `firewall_bot.py` |

**Session Breakdown:**
- **Session 07 – The Sentry:** Python foundations · virtual environments · port scanner
- **Session 08 – The Paper Trail:** Lists, loops, blacklist cross-referencing · network event categorization
- **Session 09 – The Conductor:** Functions, `try/except`, automated log parsing and blocklist generation

**Assessment:** Defensive automation script that parses system logs and generates threat intelligence reports.

</details>

<details>
<summary><b>🌐 Domain 3 · Network Traffic Analysis & Active Defense</b></summary>

<br/>

> *"The attacker leaves breadcrumbs. The defender learns to read them."*

**Core Competencies:** Deep Packet Inspection · IDS/IPS configuration · DFIR

| Week | Focus | Tools |
|:---|:---|:---|
| **W2 · Packet Analysis** | TCP 3-Way Handshake, TLS inspection, Wireshark pcap sessions | `Wireshark` · `tcpdump` |
| **W7 · Reconnaissance** | OSINT passive recon, active Nmap scanning, NSE vulnerability scripts | `Nmap` · OSINT frameworks |
| **W8 · Exploitation** | CVE analysis, CVSS scoring, Metasploit Framework exploitation | `Metasploit` |
| **W9 · Web App Security** | Burp Suite traffic interception, SQLi, XSS cookie theft | `Burp Suite` · `sqlmap` |
| **W10 · Digital Forensics** | Chain of custody, disk forensics, memory forensics | `Autopsy` · `Volatility` |
| **W11 · Active Defense** | AWS Security Groups, Suricata IDS, EDR demonstration | `Suricata` · `Snort` |

**Assessment:** Live incident response simulation — identify attack vector via packet analysis, then harden the system.

</details>

<summary><b>☁️ Domain 4 · Cloud Infrastructure & Identity Governance</b></summary>

**Core Competencies:** Secure multi-tenant AWS architecture · IAM policy engineering · data encryption

| Week | Focus | Artifact |
